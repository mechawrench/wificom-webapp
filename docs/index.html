Introduction

Base URL: https://digiunlocker-wificom-filament.test

This documentation aims to provide all the information you need to work with our API.

As you scroll, you'll see code examples for working with the API in different programming languages in the dark area to the right (or as part of the content on mobile). You can switch the language used with the tabs at the top right (or from the nav menu at the top left on mobile).

Authenticating requests

This API is authenticated by sending an Authorization header with the value "Bearer {USER_API_TOKEN}".

All authenticated endpoints are marked with a requires authentication badge in the documentation below.

You can retrieve your token by visiting your settings and clicking Generate API token at the bottom of the page.

Endpoints

POST api/v1/application/send_digirom

requires authentication

Example request:
curl --request POST \
    "https://digiunlocker-wificom-filament.test/api/v1/application/send_digirom" \
    --header "Authorization: Bearer {USER_API_TOKEN}" \
    --header "Content-Type: application/json" \
    --header "Accept: application/json" \
    --data "{
    \"device_uuid\": \"jyjtupqustudmnrv\",
    \"application_uuid\": \"tegpautokbtlqsbk\",
    \"comment\": \"jkhlvxgpapdhgpnwu\",
    \"digirom\": \"fugit\"
}"

const url = new URL(
    "https://digiunlocker-wificom-filament.test/api/v1/application/send_digirom"
);

const headers = {
    "Authorization": "Bearer {USER_API_TOKEN}",

    "Accept": "application/json",

    "application_uuid": "tegpautokbtlqsbk",

fetch(url, {
    method: "POST",

        'headers' => [
            'Authorization' => 'Bearer {USER_API_TOKEN}',
            'Content-Type' => 'application/json',
            'Accept' => 'application/json',
        ],
        'json' => [
            'device_uuid' => 'jyjtupqustudmnrv',
            'application_uuid' => 'tegpautokbtlqsbk',
            'comment' => 'jkhlvxgpapdhgpnwu',

    ]

$body = $response->getBody();
